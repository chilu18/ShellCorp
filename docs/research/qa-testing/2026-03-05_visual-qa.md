# Visual QA - Business + Ledger Tabs

## Expected UI Spec

- **Screen intent**: Team operators can understand business orchestration in `Business` and money movement in `Ledger` in under 10 seconds.
- **Components**:
  - `Business`: skill library (left), flow canvas (center/right), save + heartbeat preview controls, readiness panel.
  - `Ledger`: balance summary, funding action form (`Fund Account`, `Record Spend`), append-only transaction timeline.
- **UX goal**: user first sees slot wiring + active skill assignments, then can switch to ledger for current balance and latest credit/debit events.
- **Layout assertions**:
  - Business split layout expected width ratio near `30/70` (`x` start of left panel near `0-35%`, canvas near `30-100%`).
  - Tabs row expected at top of team panel and includes `Business` + `Ledger`.
  - Ledger summary expected above transaction list; transaction list occupies remaining vertical space with scroll.

## Fix Plan

- Add deterministic QA hooks for panel entry:
  - expose labeled controls for opening team panel (e.g. `Open Team Panel` in debug menu or data-testid bound command).

- Add accessible labels for radial menu items so browser automation can select exact entries.
- Keep geometry assertions in follow-up QA once panel access is deterministic:
  - Business split ratio and canvas occupancy.
  - Ledger summary above timeline with non-overflowing action row.
