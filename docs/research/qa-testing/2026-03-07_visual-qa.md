# Visual QA: Object UI Bindings

## Expected UI Spec
- Screen intent: let operators configure office-object runtime panels in builder mode, then open those panels in normal mode without leaking builder controls.
- Components:
  - Office scene canvas remains the primary surface.
  - Builder-selected object shows radial object actions only in builder mode.
  - Builder config opens a right-side `Object Builder` sheet with display and runtime interaction controls.
  - Runtime click opens a right-side object interaction sheet with title, external-open CTA, and iframe viewer area.
- UX goal: operators should first notice the right-side panel title/context, then the main form or runtime content, then the closing/external actions.
- Layout map:
  - Scene canvas fills background.
  - Builder/runtime sheets dock to the right edge.
  - Primary config CTA (`Save Object Config`) sits in the lower-right action zone of the builder sheet.
  - Runtime CTA (`Open In New Tab`) sits near the top-right of the runtime sheet header/body transition.
- Visual hierarchy:
  - Builder: `Object Builder` title -> embed settings form -> save button.
  - Runtime: object title -> `Open In New Tab` -> iframe area.
- Responsiveness expectation: sheet remains right-docked and usable at desktop width; scene remains visible behind it.
- Layout assertions:
  - Builder sheet title bbox pct expected: `x 70-100`, `y 0-8`, `w 15-30`, `h 2-6`.
  - Builder save CTA bbox pct expected: `x 82-100`, `y 88-100`, `w 10-18`, `h 3-6`.
  - Runtime external-open CTA bbox pct expected: `x 70-95`, `y 15-28`, `w 9-16`, `h 3-6`.
  - Builder menu gating: non-builder selected object should not add object-action buttons; builder-selected object should add radial object-action buttons.
